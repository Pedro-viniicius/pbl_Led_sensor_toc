Montagem e programação: sensor de toque com 5 LEDs

Siga as etapas com calma. O objetivo não é apenas fazer funcionar, mas entender o que cada fio, componente e linha de código faz.

Objetivo Materiais Segurança Circuito Montagem Código Explicação Testes Problemas comuns Desafios

Objetivo

Missão da aula Criar um sistema embarcado que lê um sensor de toque e acende 5 LEDs em sequência enquanto o sensor continua pressionado.

Neste projeto, o sensor de toque é a entrada, o programa no microcontrolador faz o processamento e os LEDs são a saída.

Materiais

1 placa Arduino UNO

É o microcontrolador principal deste roteiro.

1 sensor de toque

Use um módulo capacitivo comum, com pinos VCC, GND e OUT ou SIG.

5 LEDs

Podem ser da mesma cor ou cores diferentes.

5 resistores

Use 220 ohms ou 330 ohms, um para cada LED.

Protoboard

Placa de montagem sem solda.

Jumpers

Fios macho-macho para conectar tudo.

Cabo USB

Para alimentar e enviar o programa.

Computador com IDE Arduino

Onde o código será escrito e enviado.

Segurança e cuidados

Atenção Nunca ligue um LED direto no pino do Arduino sem resistor. O resistor limita a corrente e ajuda a proteger o LED e a placa.
Antes de ligar no USB Confira se 5V não está conectado direto ao GND. Esse erro pode causar aquecimento e mau funcionamento.
  • Monte o circuito com a placa desconectada do USB sempre que possível.
  • Confira a posição do LED: a perna longa costuma ser o lado positivo.
  • Todos os componentes precisam compartilhar o mesmo GND.
  • Se algo esquentar, desconecte o USB e chame o professor.

Entendendo o circuito

O sensor de toque envia um sinal digital para o Arduino. O Arduino lê esse sinal no pino 2. Depois, o Arduino liga ou desliga os LEDs usando os pinos 4 a 8.

Passo a passo da montagem

Desconecte o Arduino do USB

Faça a montagem sem energia para evitar curtos acidentais.

Coloque os 5 LEDs na protoboard

Deixe espaço entre eles. A perna longa é o ânodo, lado que receberá o sinal do pino digital através do resistor.

Conecte um resistor para cada LED

Ligue cada pino digital do Arduino ao resistor e depois ao lado positivo do LED. Cada LED precisa do seu próprio resistor.

Ligue o lado negativo dos LEDs ao GND

A perna curta do LED deve ir para o trilho de GND da protoboard.

Conecte os pinos dos LEDs ao Arduino

LED 1 no pino 4, LED 2 no pino 5, LED 3 no pino 6, LED 4 no pino 7 e LED 5 no pino 8.

Conecte o sensor de toque

VCC no 5V, GND no GND e OUT ou SIG no pino digital 2.

Confira o GND comum

O GND do sensor e o GND dos LEDs devem estar conectados ao GND do Arduino.

Conecte o USB

Depois da conferência, conecte o Arduino ao computador.

Código do projeto

Versão simples para entender a ideia Esta versão usa delay(). Ela é fácil de entender, mas deixa o programa parado durante a pausa. 
const int pinoSensor = 2;
const int pinosLeds[] = {4, 5, 6, 7, 8};
const int quantidadeLeds = 5;

void setup() {
  pinMode(pinoSensor, INPUT);

  for (int i = 0; i < quantidadeLeds; i++) {
    pinMode(pinosLeds[i], OUTPUT);
  }
}

void loop() {
  int estadoSensor = digitalRead(pinoSensor);

  if (estadoSensor == HIGH) {
    for (int nivel = 1; nivel <= quantidadeLeds; nivel++) {
      atualizarLeds(nivel);
      delay(600);

      if (digitalRead(pinoSensor) == LOW) {
        break;
      }
    }
  } else {
    atualizarLeds(0);
  }
}

void atualizarLeds(int nivel) {
  for (int i = 0; i < quantidadeLeds; i++) {
    if (i < nivel) {
      digitalWrite(pinosLeds[i], HIGH);
    } else {
      digitalWrite(pinosLeds[i], LOW);
    }
  }
}
Versão principal recomendada Esta versão usa millis() para controlar o tempo sem travar o programa com pausas longas. 
const int pinoSensor = 2;
const int pinosLeds[] = {4, 5, 6, 7, 8};
const int quantidadeLeds = 5;

int nivelAtual = 0;
unsigned long ultimoAvanco = 0;
const unsigned long intervaloAvanco = 600;

void setup() {
  pinMode(pinoSensor, INPUT);

  for (int i = 0; i < quantidadeLeds; i++) {
    pinMode(pinosLeds[i], OUTPUT);
    digitalWrite(pinosLeds[i], LOW);
  }
}

void loop() {
  int estadoSensor = digitalRead(pinoSensor);

  if (estadoSensor == HIGH) {
    unsigned long agora = millis();

    if (agora - ultimoAvanco >= intervaloAvanco) {
      ultimoAvanco = agora;

      if (nivelAtual < quantidadeLeds) {
        nivelAtual++;
      }
    }
  } else {
    nivelAtual = 0;
    ultimoAvanco = millis();
  }

  for (int i = 0; i < quantidadeLeds; i++) {
    if (i < nivelAtual) {
      digitalWrite(pinosLeds[i], HIGH);
    } else {
      digitalWrite(pinosLeds[i], LOW);
    }
  }
}

Como enviar o código

Abra a IDE Arduino

Crie um novo arquivo.

Cole a versão principal do código

Confira se nada ficou faltando, principalmente chaves, parênteses e ponto e vírgula.

Selecione a placa

No menu de placas, escolha Arduino UNO. Se estiver usando ESP32, escolha o modelo correto.

Selecione a porta

Escolha a porta em que a placa apareceu depois de conectar o USB.

Clique em verificar

Esse botão confere se há erro de escrita no código.

Clique em enviar

A IDE compila e grava o programa no microcontrolador.

Explicação do código por blocos

Constantes e pinos

const int pinoSensor = 2; define que o sensor está no pino digital 2. const indica que esse valor não deve mudar durante o programa.

pinosLeds[] é uma lista. Uma lista guarda vários valores com o mesmo nome. Aqui, ela guarda os pinos dos 5 LEDs.

Variáveis de estado

nivelAtual guarda quantos LEDs devem estar acesos. Isso é o estado do sistema.

ultimoAvanco guarda o último momento em que o nível aumentou. intervaloAvanco define a velocidade da sequência.

setup()

Essa função roda uma vez quando a placa liga. Ela configura o sensor como entrada e os LEDs como saída.

loop()

Essa função repete para sempre. Primeiro ela lê o sensor com digitalRead. Depois usa if para decidir o que fazer.

Condição if

Se o sensor estiver em HIGH, o programa verifica o tempo e pode aumentar nivelAtual. Senão, o nível volta para zero.

Laço for

O for percorre todos os LEDs, do primeiro ao quinto. Para cada LED, o programa decide se ele fica ligado ou desligado.

Erro comum de programação O computador segue instruções exatas. Se o pino no código for 5, mas o fio estiver no pino 6, o programa não “adivinha” o erro.

Conceitos de eletrônica usados

Tensão

É como pressão. No Arduino UNO, usamos 5V para alimentar o sensor.

Corrente

É o fluxo de cargas. Se for grande demais, pode danificar componentes.

Resistência

É a dificuldade para a corrente passar. O resistor aumenta essa dificuldade de forma controlada.

GND

É a referência comum do circuito. Não é apenas “um fio qualquer”; é a base usada para medir os sinais.

VCC

É o ponto de alimentação positiva do módulo, como 5V ou 3V3.

Sinal

É a informação elétrica que representa algo, como sensor pressionado ou solto.

Lista de problemas comuns

Nenhum LED acende

Confira GND, pinos, resistores, posição dos LEDs e se o código foi enviado.

Um LED não acende

Confira se ele está invertido, queimado ou no pino errado.

Sensor não responde

Confira VCC, GND, OUT e se o sensor realmente trabalha com a tensão usada. Se o comportamento ficar invertido, o módulo pode enviar LOW ao tocar; nesse caso, peça ajuda para inverter a condição no código.

LEDs acendem fora de ordem

Confira se os fios dos LEDs batem com a lista {4, 5, 6, 7, 8}.

Erro na IDE

Procure ponto e vírgula faltando, chaves trocadas, placa errada ou porta errada.

Confusão com tensão e corrente

Tensão empurra. Corrente flui. Resistência limita o fluxo.

Desafios de extensão

  1. Mude intervaloAvanco para deixar a sequência mais rápida ou mais lenta.
  2. Adicione mais LEDs e atualize a lista de pinos.
  3. Faça os LEDs apagarem um por um quando o sensor for solto.
  4. Adicione um buzzer que toque quando todos os LEDs estiverem acesos.
  5. Transforme o projeto em uma “barra de volume” controlada por toque.
  6. Use o Monitor Serial para imprimir o valor de nivelAtual.

Adaptação para ESP32

Atenção O ESP32 trabalha com lógica de 3,3V. Não coloque 5V em um pino de entrada do ESP32.

Para ESP32, escolha pinos digitais disponíveis na sua placa e alimente o sensor em 3V3 se ele aceitar essa tensão. Depois, altere no código os números dos pinos do sensor e dos LEDs.
