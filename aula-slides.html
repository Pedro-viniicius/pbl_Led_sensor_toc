PBL: Sensor de Toque + LEDs Sequenciais

Desafio: construir um sistema embarcado em que o tempo de toque aumenta o nível de LEDs acesos.

Ao final da aula O estudante deve conseguir explicar entrada, processamento, saída, variável, condição, repetição, estado, tensão, corrente, resistência e resistor no LED.

Situação-problema

Um sistema embarcado é um pequeno computador dentro de um objeto maior. Ele lê sinais, toma decisões e controla alguma saída.

Botão de elevador
Micro-ondas
Teclado
Painel de alarme
Lâmpada inteligente
Pausa para discussão Onde há entrada, processamento e saída em cada exemplo?

Desafio do projeto

  • Ao tocar no sensor, o primeiro LED acende.
  • Se o toque continua, os LEDs acendem em sequência.
  • Os LEDs funcionam como uma barra de nível.
  • Ao soltar o sensor, nesta versão, o nível volta para zero.

Eletrônica básica

Tensão

Diferença de potencial elétrico. Ajuda a empurrar a corrente.

Corrente

Fluxo de cargas elétricas passando pelo circuito.

Resistência

Dificuldade para a corrente passar.

Atenção O LED precisa de resistor. Sem resistor, a corrente pode ficar alta demais e queimar o LED ou danificar a placa.

VCC é alimentação. GND é a referência comum do circuito. Sinal é a informação que entra ou sai por um pino.

Analogia da água

Tensão

Como a pressão da água.

Corrente

Como a quantidade de água fluindo.

Resistência

Como um cano mais estreito.

Resistor

Limita o fluxo para o LED não receber corrente demais.

Conceito-chave O resistor não é enfeite. Ele controla o quanto de corrente passa pelo LED.

Programação básica

Um programa é uma receita com instruções exatas. O computador não adivinha intenção: ele segue o que foi escrito.

Entrada

Informação que entra: sensor de toque.

Processamento

Decisão feita pelo código: aumentar ou zerar o nível.

Saída

Ação no mundo físico: acender LEDs.

No Arduino, a função loop() repete para sempre enquanto a placa está ligada.

Variáveis

Uma variável é como uma caixa com etiqueta. Ela guarda um valor que o programa usa.

int nivelAtual = 0;
int estadoSensor = LOW;
unsigned long ultimoTempo = 0;
Exemplo no projeto nivelAtual guarda quantos LEDs devem estar acesos neste momento.

Condições

if significa “se”. else significa “senão”. É uma decisão.

if (estadoSensor == HIGH) {
  // sensor pressionado
} else {
  // sensor solto
}
Erro comum O computador não entende “mais ou menos”. A comparação precisa resultar em verdadeiro ou falso.

Repetição

for

Usamos quando sabemos quantas vezes queremos repetir.

for (int i = 0; i < 5; i++) {
  // repete 5 vezes
}

while

Usamos quando queremos repetir enquanto algo for verdadeiro.

while (sensorPressionado) {
  // repete enquanto for verdade
}

Entrada e saída digital

digitalRead

Lê um pino digital. Resultado: HIGH ou LOW.

digitalWrite

Escreve em um pino digital. Liga ou desliga uma saída.

HIGH

Nível alto. Em geral, ligado ou com sinal.

LOW

Nível baixo. Em geral, desligado ou sem sinal.

Sensor é entrada. LED é saída.

Estratégia de programação

  1. Ler o sensor de toque.
  2. Se estiver pressionado, esperar um intervalo de tempo.
  3. Aumentar o nível de LEDs acesos.
  4. Atualizar todos os LEDs com um for.
  5. Se soltar, zerar o nível.
Estado O valor de nivelAtual representa a memória momentânea do sistema.

Código completo

const int pinoSensor = 2;
const int pinosLeds[] = {4, 5, 6, 7, 8};
const int quantidadeLeds = 5;

int nivelAtual = 0;
unsigned long ultimoAvanco = 0;
const unsigned long intervaloAvanco = 600;

void setup() {
  pinMode(pinoSensor, INPUT);

  for (int i = 0; i < quantidadeLeds; i++) {
    pinMode(pinosLeds[i], OUTPUT);
    digitalWrite(pinosLeds[i], LOW);
  }
}

void loop() {
  int estadoSensor = digitalRead(pinoSensor);

  if (estadoSensor == HIGH) {
    unsigned long agora = millis();

    if (agora - ultimoAvanco >= intervaloAvanco) {
      ultimoAvanco = agora;

      if (nivelAtual < quantidadeLeds) {
        nivelAtual++;
      }
    }
  } else {
    nivelAtual = 0;
    ultimoAvanco = millis();
  }

  for (int i = 0; i < quantidadeLeds; i++) {
    if (i < nivelAtual) {
      digitalWrite(pinosLeds[i], HIGH);
    } else {
      digitalWrite(pinosLeds[i], LOW);
    }
  }
}

Leitura do código

const int

Guarda pinos e valores que não mudam durante o programa.

Vetor

pinosLeds[] é uma lista de pinos dos LEDs.

setup()

Configura entradas e saídas uma vez.

loop()

Repete a leitura, decisão e atualização.

millis()

Conta o tempo sem travar o programa com uma pausa longa.

for

Percorre todos os LEDs e decide quais ficam acesos.

Testes

  • Sem tocar: todos os LEDs devem ficar apagados.
  • Toque curto: apenas um LED pode acender rapidamente.
  • Toque longo: os LEDs acendem um por um.
  • Toque muito longo: todos os LEDs ficam acesos.
  • Ao soltar: todos apagam.
Teste rápido Peça que um estudante explique o que mudou em nivelAtual durante o toque.

Depuração: erros comuns

LED invertido: perna longa e perna curta foram trocadas.
Resistor ausente: o LED não está protegido.
GND errado: sensor e LEDs não compartilham referência.
Pino errado: circuito e código não batem.
Sensor alimentado incorretamente: VCC e GND foram trocados.
Sensor com lógica invertida: alguns módulos enviam LOW quando são tocados.
Placa ou porta errada na IDE Arduino.
Ponto e vírgula esquecido no código.
Confusão entre intenção humana e instrução exata.
